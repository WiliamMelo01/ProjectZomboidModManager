# Contribuição

Siga estas regras básicas ao contribuir com o projeto:

- Faça commits pequenos e atômicos após cada alteração significativa.
- Mensagens de commit devem ser claras e padronizadas (ver `./.gitmessage.txt`).
- Abra issues para bugs grandes ou discussões de design antes de grandes mudanças.
- Inclua testes ou passos para reproduzir quando reportar bugs.

Padrões de mensagem de commit (exemplos):

- `feat(server): detectar instâncias do servidor automaticamente`
- `fix(scanner): corrigir leitura de workshop id`
- `chore: atualizar dependências de dev`

Obrigado por contribuir!
